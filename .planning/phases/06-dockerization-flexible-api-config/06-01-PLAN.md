# Phase 06 - Plan 01: Complete Dockerization & One-Command Setup

## 🎯 Objective
Containerize the ENTIRE stack — Frontend, Backend, Database, and Bifrost AI Gateway — into a single `docker compose up` deployment. The user provides their Groq API key (required) and optionally their Langfuse Cloud keys for AI tracing. No image rebuilds needed for config changes.

## 📐 System Architecture (Dockerized)

```
User's Browser
     │
     ├──→ http://localhost:3000  →  [frontend container]  (Next.js UI)
     │                                      │
     │                                      │ NEXT_PUBLIC_API_URL
     │                                      ▼
     └──→ http://localhost:3500  →  [backend container]   (NestJS API)
                                            │
                              ┌─────────────┼──────────────┐
                              │             │              │
                              ▼             ▼              ▼
                    [db container]   [bifrost container]  Langfuse Cloud
                    PostgreSQL:5432  Gateway:8080         (optional)
                                         │
                                         ▼
                                   Groq Cloud API
                              (user's API key)
```

### Port Map (exposed to user's machine)
| Port | Service | What the user sees |
|------|---------|-------------------|
| `3000` | Frontend | Dashboard UI |
| `3500` | Backend | API (browser calls) |
| `8080` | Bifrost | AI Gateway Web UI (manage providers) |

### Internal-only (not exposed)
| Port | Service | Internal hostname |
|------|---------|------------------|
| `5432` | PostgreSQL | `db:5432` |

### Step 4: Advanced Configuration (optional)
```
Configure advanced features? (y/n) [n]: █
```
If yes, prompt for:
- Google OAuth (Client ID + Secret) — for Google login
- GitHub OAuth (Client ID + Secret) — for GitHub login
- AWS S3 (Access Key + Secret + Bucket) — for file uploads
- Custom DB password — for security hardening

### Step 5: Auto-Configuration (no user input)
The script automatically:
1. Copies `.env.example` → `.env`
2. Generates a random `JWT_SECRET` via `openssl rand -base64 32`
3. Injects the user's Groq API key into the Bifrost config (`bifrost-data/config.json`)
4. Creates a Bifrost virtual key and writes it to `BIFROST_API_KEY` in `.env`
5. Sets all internal Docker hostnames (`DB_HOST=db`, `BIFROST_BASE_URL=http://bifrost:8080/v1`)
6. Writes Langfuse keys to `.env` if provided (or leaves blank for graceful fallback)

### 6. Create Cross-Platform Setup Scripts
<read_first>

<action>
**6a. Create `setup.sh` (Mac/Linux):**
1. Check Docker + Docker Compose installed. If not → print install link and exit.
2. Check if Docker daemon is running (`docker info`). If not → prompt user to start Docker.
3. Check if `.env` exists. If yes → ask "Config exists. Reconfigure? (y/n)".
4. Copy `.env.example` → `.env`.
5. Prompt: "Enter your Groq API key:" (required, no default).
6. Auto-generate `JWT_SECRET` via `openssl rand -base64 32`.
7. Generate Bifrost `config.json` in `./bifrost-data/` with Groq provider and the user's API key. Create a virtual key and write it to `BIFROST_API_KEY` in `.env`.
8. Prompt: "Configure advanced features? (y/n)" — if yes, prompt for OAuth, AWS.
9. Use `sed` to replace placeholder values in `.env` (use `|` delimiter to avoid URL issues, escape special chars in API keys).
10. Prompt: "Start the dashboard now? (y/n) [y]" → `docker compose pull && docker compose up -d`.
11. Print success with URLs for all 4 services.

**6b. Create `setup.ps1` (Windows PowerShell):**
Equivalent logic using:
- `Get-Command docker` for detection
- `Read-Host` for prompts
- `(Get-Content .env) -replace` for substitution
- `[System.Security.Cryptography.RandomNumberGenerator]` for JWT secret

**Edge cases to handle:**
- `sed -i` Mac vs Linux difference → use `sed -i.bak` then delete `.bak`
- API keys with special chars (`&`, `/`) → escape for sed
- User runs setup twice → detect and offer reconfigure
- Docker daemon not running → separate check from Docker installed
</action>
<acceptance_criteria>
- `setup.sh` runs on Mac and Linux without errors.
- `setup.ps1` runs on Windows PowerShell without errors.
- Both scripts detect Docker, create `.env`, configure Bifrost, and launch the stack.
